# Сводка требований из PDF (55 страниц)

## Основные требования

### Архитектура
- **Многослойная архитектура** с разделением:
  - Бизнес-объекты (модели/домен)
  - Сценарии взаимодействия (бизнес-логика)
  - Слой представления (с **Презентером или ViewModel**)
  - Фреймворки (UI, БД, сеть)
- **Правило зависимостей**: внутренние слои не знают о внешних
- Файлы распределены по папкам согласно архитектуре

### Технические требования
- Минимальная версия Android: **11.0** (API 30)
- Бэкенд: **Supabase**
- Git для контроля версий
- Адаптивная верстка (разные размеры экранов)
- Обработка ошибок с диалогами
- Экран "No Connection" при отсутствии интернета
- Индикация загрузки запросов
- Устойчивость к сбоям
- Комментирование кода (описание назначения класса)

### Экраны

## ЛАБОРАТОРНАЯ РАБОТА №1

1. **Экран загрузки (Splash)**
   - Название, логотип, слоган
   - Отображение 2-3 секунды
   - Переход на Onboarding (первый раз) или Main (если есть токен)

2. **Нет подключения к интернету**
   - Иконка, сообщение об ошибке
   - Кнопка "Повторить попытку"

3. **Приветствие (Onboarding)**
   - Несколько слайдов с преимуществами
   - Кнопки: "Пропустить", "Далее", "Поехали"
   - Сохранение факта прохождения

4. **Входа/регистрации (Getting Started)**
   - Название, логотип, слоган
   - Кнопки: "Войти", "Зарегистрироваться"

5. **Вход в аккаунт (Login)**
   - Поля: Email (с валидацией), Пароль (показать/скрыть)
   - Кнопки: "Войти", "Войти через Google", "Зарегистрироваться"
   - Сохранение токена и сессии

6. **Регистрация (шаг 1)**
   - Email, Пароль, Повторить пароль
   - Чекбокс согласия с условиями
   - Валидация всех полей

7. **Регистрация (шаг 2)**
   - Фамилия, Имя, Отчество, Дата рождения, Пол
   - Валидация обязательных полей

8. **Регистрация (шаг 3)**
   - Фото профиля (необязательно)
   - Номер водительского удостоверения
   - Дата выдачи
   - Загрузка фото документов

9. **Успешная регистрация**
   - Поздравление
   - Кнопка "Далее" → Главный экран

## ЛАБОРАТОРНАЯ РАБОТА №2

10. **Настройки**
    - Профиль пользователя (аватар, имя, email)
    - Меню: Мои бронирования, Тема, Уведомления, Подключить автомобиль, Помощь, Пригласи друга
    - Сохранение настроек после перезапуска

11. **Профиль**
    - Отображение/изменение аватара
    - Кнопка "Выйти из профиля" → Экран входа

12. **Главная (Main)**
    - Список доступных автомобилей (загрузка с сервера)
    - Поиск по марке → Экран "Результаты поиска"
    - Кнопки: "Забронировать", "Детали"
    - Индикатор загрузки и обработка ошибок

13. **Результаты поиска**
    - Список найденных автомобилей
    - Кнопки: "Забронировать", "Детали"
    - Обновление при изменении запроса

## ЛАБОРАТОРНАЯ РАБОТА №3

14. **Детали автомобиля**
    - Полная информация об автомобиле
    - Кнопка "Забронировать" → Экран "Оформление аренды"
    - Иконка "сердце" для добавления в избранное
    - Кнопка "Назад"

15. **Оформление аренды**
    - Информация об автомобиле
    - Выбор дат начала и конца аренды
    - Адрес местоположения
    - Расчет стоимости (аренда + страховка + депозит)
    - Кнопка "Продолжить" → Экран "Удачной поездки"

16. **Удачной поездки (Успешное бронирование)**
    - Сообщение об успешном бронировании
    - Кнопки: "Перейти к своим бронированиям", "Домой"

17. **Мои бронирования**
    - Список активных и завершенных бронирований
    - Нажатие на активное → Экран "Детальная информация о бронировании"

18. **Детальная информация о бронировании**
    - Номер бронирования
    - Полная информация об автомобиле и аренде
    - ФИО водителя, номер водительского удостоверения
    - Статус аренды
    - Расчет стоимости
    - Кнопка "Отменить бронирование" (для активных)

19. **Избранное**
    - Список избранных автомобилей
    - Кнопки: "Забронировать", "Детали"

## ЛАБОРАТОРНАЯ РАБОТА №4

20. **Стать арендодателем**
    - Информационные блоки
    - Кнопка "Начать" → Экран "Добавление автомобиля (шаг 1)"

21. **Добавление автомобиля (шаг 1)**
    - Поле ввода адреса
    - Кнопка "Далее" (активна после заполнения) → Шаг 2

22. **Добавление автомобиля (шаг 2)**
    - Год выпуска, Марка, Модель
    - Тип трансмиссии (выпадающий список)
    - Пробег, Описание
    - Кнопка "Отправить" (активна после заполнения всех полей) → Шаг 3

23. **Добавление фотографий автомобиля**
    - Загрузка до 5 фотографий
    - Кнопка "Далее" (активна после добавления фото) → Шаг 4

24. **Успешное добавление автомобиля**
    - Сообщение об успешном добавлении
    - Кнопка "На главную"

## Анализ папки zavar

В папке `zavar/app/src/main/` есть:
- ✅ Готовые layouts для всех экранов
- ✅ Activity классы
- ✅ Data классы (Car, User, Booking, Settings)
- ✅ Utils (Validator, NetworkUtils, Prefs, ImagePicker)
- ✅ Структура с разделением на папки

**Можно использовать layouts из zavar**, но нужно:
1. Проверить соответствие дизайну из Figma
2. Адаптировать под текущую структуру проекта
3. Добавить архитектуру (Presenter/ViewModel)
4. Интегрировать Supabase
5. Добавить обработку ошибок
